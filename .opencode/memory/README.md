# OpenCode Experience Memory

This folder stores reusable engineering experience for this project/team. It is intentionally plain Markdown so agents can search it quickly with `rg` and humans can review it in Git.

Recommended use:

1. Before fixing: search `solution-index.md`, `success-ledger.md`, and `failure-ledger.md`.
2. During fixing: append attempt summaries to `decision-log.md`.
3. After fixing: append successes/failures and update the index.
4. Periodically run `/knowledge-sync` to improve tags.

Do not store secrets, private credentials, customer data, or full proprietary logs here.
