# /verify — Verification Gate

Run the most relevant verification for the current project.

Checklist:
- Identify the correct test/build/lint command from repo files.
- Run the smallest meaningful verification first.
- If it fails, capture exact error and likely cause.
- If no automated test exists, perform a deterministic manual check and explain the limitation.

Output:
- Commands run
- Results
- Files/areas verified
- Remaining risk
