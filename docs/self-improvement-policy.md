# Self-Improvement Policy

The team may improve its local workflow memory, but it must not silently change product requirements.

## Allowed Self-Improvement

- Append success/failure records.
- Add searchable tags to solution index.
- Add generalized lessons to patterns.
- Add project-specific gotchas to documentation.
- Propose a change to `AGENTS.md` or skill instructions when repeated evidence supports it.

## Not Allowed Without User Approval

- Changing project architecture just because a prior task did.
- Installing new services or MCP servers.
- Sending private data to external tools.
- Editing global OpenCode config outside the project.
- Removing safety permissions.

## Retrospective Questions

At the end of a task, answer:

1. What exact method worked?
2. What exact method failed?
3. What should be tried first next time?
4. What should be avoided next time?
5. Which file was updated with the lesson?
